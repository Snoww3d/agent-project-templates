# Agent Project Templates

Templates for scaffolding new projects with Claude Code agent capabilities.

## Overview

This repository contains Handlebars-style templates that generate:

- **CLAUDE.md** - Central quick-reference for Claude Code
- **Workflows** - Development workflows (feature, bug, tech-debt)
- **Commands** - Planning commands for discovery phase
- **Agents** - Custom specialized agents (code review)
- **Documentation** - Standards, architecture, roadmap, tech debt tracking

## Installation

Install the skill to your user-level Claude Code commands:

```bash
# Clone the repo
git clone https://github.com/Snoww3d/agent-project-templates.git
cd agent-project-templates

# Copy the skill to your Claude commands directory
mkdir -p ~/.claude/commands
cp skill/create-agent-project.md ~/.claude/commands/
```

## Usage

Once installed, run the skill in any Claude Code session:

```bash
/create-agent-project
```

The skill will:
1. Ask questions about your project
2. Clone this repository (templates)
3. Process templates with your answers
4. Generate customized files in your project

## Template Syntax

Templates use Handlebars-style placeholders:

### Variables

```handlebars
{{PROJECT_NAME}}
{{PROJECT_DESCRIPTION}}
```

### Conditionals

```handlebars
{{#if frontend}}
Frontend-specific content here
{{/if}}

{{#if docker}}
Docker content
{{else}}
Non-docker content
{{/if}}
```

### Iteration

```handlebars
{{#each services}}
- {{this.name}}: http://localhost:{{this.port}}
{{/each}}
```

## Context Object

See `schema.json` for the full schema of values available to templates.

Key values:
- `PROJECT_NAME`, `PROJECT_ROOT`, `PROJECT_DESCRIPTION`
- `frontend`, `backend`, `python`, `docker` (booleans)
- `FRONTEND_DIR`, `BACKEND_DIR`, etc. (paths)
- `*_CMD` values for various commands
- `github` or `gitlab` for git platform
- `services` array for Docker services

## Customization

After scaffolding, customize the generated files for your project:

1. **CLAUDE.md** - Add project-specific quick reference
2. **Workflows** - Adjust quality checks and verification steps
3. **Documentation** - Fill in architecture details, roadmap phases

## Contributing

To add new templates or improve existing ones:

1. Add/edit `.hbs` files in `templates/`
2. Update `schema.json` if adding new context variables
3. Test with the `/create-agent-project` skill

## Version

Generated files include a version comment:
```markdown
<!-- Generated by agent-scaffold v1.0 -->
```

This helps track which version was used for initial scaffolding.
